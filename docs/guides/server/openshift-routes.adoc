== Overview

When running {project_name} on OpenShift, the Admin API and Admin Console should
not be accessible from external callers. This guide explains how to configure
OpenShift Routes with HAProxy to allow or block connections based on URL path
and IP address.

NOTE: URL-based filtering only works with *TLS reencrypt* routes. TLS passthrough
routes cannot inspect or filter by URL path because the traffic is not terminated
at the router.

== Exposed path recommendations

The following paths should be exposed to the public internet:

* `/realms/` — required for authentication flows
* `/resources/` — required for serving static assets
* `/robots.txt` — search engine guidance

The following paths should be blocked from external access:

* `/admin/` — Admin Console and Admin REST API
* `/metrics` — internal metrics endpoint
* `/health` — internal health endpoint
* `/management` — management interface

== Configuring a TLS reencrypt Route

A TLS reencrypt Route decrypts traffic at the HAProxy router, applies filtering
rules, and re-encrypts it before forwarding to {project_name}.

=== Block the Admin API by URL path

[source,yaml]
----
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: keycloak-admin-internal
  annotations:
    haproxy.router.openshift.io/ip_whitelist: "10.0.0.0/8 192.168.0.0/16" <1>
spec:
  host: keycloak.example.com
  path: /admin <2>
  to:
    kind: Service
    name: keycloak
  tls:
    termination: reencrypt
    insecureEdgeTerminationPolicy: Redirect
----
<1> Replace with your internal IP ranges. Only these IPs may access `/admin`.
<2> This route matches only the `/admin` path.

=== Allow public access to authentication paths

[source,yaml]
----
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: keycloak-public
spec:
  host: keycloak.example.com
  path: /realms <1>
  to:
    kind: Service
    name: keycloak
  tls:
    termination: reencrypt
    insecureEdgeTerminationPolicy: Redirect
----
<1> Repeat for `/resources/` and `/robots.txt` as needed.

== IP allowlisting with HAProxy annotations

OpenShift HAProxy supports the `haproxy.router.openshift.io/ip_whitelist`
annotation to restrict access by source IP address or CIDR range.

[source,yaml]
----
metadata:
  annotations:
    haproxy.router.openshift.io/ip_whitelist: "10.0.0.0/8"
----

Multiple ranges are space-separated. Traffic from IPs not in the list is
dropped with a 403 response.

== Limitations

* IP allowlisting and URL filtering are only available for *TLS reencrypt*
  and *edge* termination. They are *not* available for *TLS passthrough*.
* Passthrough routes forward raw TLS traffic directly to the pod — HAProxy
  cannot inspect the request URL or apply ACLs.

|===
| TLS Mode       | IP filtering | URL filtering

| Edge           | ✓            | ✓
| Reencrypt      | ✓            | ✓
| Passthrough    | ✓            | ✗
|===
