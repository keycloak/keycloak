[[_enabling_scim_]]
= Enabling SCIM
[role="_abstract"]

Enabling the SCIM APIs in {project_name} requires two steps: enabling the feature at the server level and then enabling it
for each realm where you want to expose the SCIM endpoints.

== Enabling the SCIM feature on the server

The SCIM API is a tech preview feature and must be explicitly enabled when starting {project_name}. To enable it, start the server
with the `scim-api` feature flag:

[source,bash]
----
bin/kc.sh start --features=scim-api
----

For development mode:

[source,bash]
----
bin/kc.sh start-dev --features=scim-api
----

== Enabling SCIM for a realm

Once the feature is enabled on the server, you must also enable SCIM for each realm individually. This can be done through
the Admin Console or the Admin REST API.

=== Using the Admin Console

.Procedure
. Click *Realm settings* in the menu.
. On the *General* tab, locate the *SCIM API* toggle.
. Set the toggle to *On*.
. Click *Save*.

.Enabling SCIM API in realm settings
image:images/scim-realm-settings.png[alt="Enabling SCIM API in realm settings"]

=== Using the Admin REST API

You can also enable SCIM for a realm programmatically through the Admin REST API:

[source,bash]
----
curl -X PUT http://localhost:8080/admin/realms/myrealm \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"scimApiEnabled": true}'
----

== Verifying the configuration

Once the SCIM API is enabled at both the server and realm levels, you can verify that it is active by querying the
Service Provider Configuration endpoint.

NOTE: All SCIM endpoints require a bearer token. See <<_scim_authorizing_access_,Authorizing access to SCIM endpoints>>
for details on how to set up a client and obtain an access token.

[source,bash]
----
curl http://localhost:8080/realms/myrealm/scim/v2/ServiceProviderConfig \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Accept: application/scim+json"
----

A successful response returns the capabilities of the SCIM service provider, including supported operations such as
`patch`, `filter`, and the authentication schemes:

[source,json]
----
{
  "schemas": ["urn:ietf:params:scim:schemas:core:2.0:ServiceProviderConfig"],
  "patch": {
    "supported": true
  },
  "bulk": {
    "supported": false,
    "maxOperations": 0,
    "maxPayloadSize": 0
  },
  "filter": {
    "supported": true,
    "maxResults": 100
  },
  "changePassword": {
    "supported": false
  },
  "sort": {
    "supported": false
  },
  "etag": {
    "supported": false
  },
  "authenticationSchemes": [
    {
      "type": "oauthbearertoken",
      "name": "OAuth Bearer Token",
      "description": "Authentication scheme using the OAuth Bearer Token standard"
    }
  ]
}
----
